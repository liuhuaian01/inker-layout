# 墨客排版 — 微信公众号编辑器

## 概述

生成可在微信客户端中正确渲染的公众号文章。微信富文本环境有严格限制：不支持外部 CSS/JS、HTML 标签受限、只能用内联样式、JavaScript 完全被禁。本 Skill 提供排版规则、参考文档、模板和校验工具，在这些约束下交付精美的交互式内容。

## 工作流决策树

收到排版请求后，按以下决策流程：

```
请求 → 识别任务类型：
├── 纯排版 → 加载 references/wechat-layout-guide.md → 应用规则 → 输出 HTML
├── 视觉/品牌设计 → 加载 references/wechat-design-tokens.md → 选模板 → 输出 HTML
├── SVG 交互 → 加载 references/wechat-svg-guide.md → 选 SVG 模板 → 输出 HTML
├── 完整文章（排版+视觉+SVG）→ 加载三份参考文档 → 组合 → 校验 → 输出
└── 仅校验 → 运行 scripts/validate_mp.py 检查已有 HTML
```

交付前务必运行 `scripts/validate_mp.py`，除非用户明确跳过。

## 核心能力

### 1. 内容排版

微信编辑器会剥离大部分 HTML/CSS。所有布局必须使用内联样式和受限标签集。

**排版流程：**
1. 阅读 `references/wechat-layout-guide.md` 了解完整约束清单
2. 将用户内容转换为微信兼容的 HTML（全内联样式）
3. 应用排版规则：仅系统字体、定义的标题层级、合适的行高
4. 处理图片：仅托管 URL、推荐宽度、`margin: 0 auto` 居中
5. 用 `scripts/validate_mp.py` 校验

**常见布局模式：**
- **长图文**：单张长图或多段图片纵向堆叠
- **卡片布局**：带圆角、阴影、内边距的 `div` 块
- **双栏**：基于 `table` 的布局（微信中唯一可靠的多栏方案）
- **问答/访谈**：左侧 accent 色边框 + 背景色的 Q&A 块
- **指南/教程**：带图标的编号步骤

### 2. 视觉设计与风格体系

十种精心设计的风格，每种包含完整配色、字体方案和布局惯例。详见 `references/wechat-design-tokens.md`。

**风格选择流程：**
1. 阅读 `references/wechat-design-tokens.md` 了解完整令牌系统
2. 判断文章的调性和内容类型
3. 从下方目录匹配最合适的风格
4. 若用户选择"描述你想要的氛围"→ 解析氛围 → 匹配后确认
5. 若用户指定品牌色 → 提取品牌色，映射到最接近主题，覆盖 accent 色
6. 选择模板：纯图文（`article-{style}.html`）或图文+SVG 组合（`article-{style}-svg.html`）
7. 自定义颜色、字体、间距和 SVG 内容
8. 输出完整自包含 HTML（全部内联样式）

**风格快速参考：**

| 风格 | 调性 | 适用 | 关键特征 |
|---|---|---|---|
| 时尚 | 杂志编辑 | 奢品、美妆、穿搭 | 高对比度、宽间距、crimson accent |
| 潮流 | 活力、Z世代 | 潮玩、音乐、趋势 | 渐变色块、霓虹紫/青、大 emoji |
| 艺术 | 画廊、精致 | 展览、摄影、散文 | 衬线字体、戏剧性留白、1px 画框 |
| 国风 | 传统、文人 | 历史、文化、诗词、非遗 | 宣纸色底、朱砂印章、楷体标题 |
| 极简 | 干净、严肃 | 深度分析、长文、文档 | 单色盘、零装饰、内容优先 |
| 科技 | 结构化、数据 | SaaS、产品、数据报告 | 统计栏、清单、蓝色渐变 |
| 编辑 | 温暖、可读 | 通用内容、故事叙述 | 默认选择、暖色 accent、卡片布局 |
| 品牌 | 公告发布 | 新品上市、活动 | Hero 横幅、渐变 CTA、品牌色 |
| 生活 | 舒适、个人 | 美食、旅行、日记 | 暖中性色、柔和渐变、圆角 |
| 奢华 | 高端 | 高端品牌、VIP 内容 | 金色 accent、暗色渐变、精致 |

**纯图文模板（`assets/templates/`）：**

布局型（任意风格可套用）：
- `article-basic.html` — 通用编辑文章
- `article-brand.html` — Hero 横幅 + 公告布局
- `article-card.html` — 卡片网格内容布局
- `article-interview.html` — 问答/访谈格式

风格专属（开箱即用，替换 `{{PLACEHOLDER}}`）：
- `article-fashion.html` — 全宽 hero、居中引用、图片注解
- `article-trendy.html` — 渐变 hero、高光块、趋势卡片
- `article-artistic.html` — 衬线字体、带框图片、装饰分隔线

**风格+SVG 组合模板（`assets/templates/`）：**

每种结合一种视觉风格与一个交互 SVG 组件：
- `article-fashion-svg.html` — 时尚 × SVG 轮播（lookbook 幻灯片）
- `article-trendy-svg.html` — 潮流 × SVG 点击揭示（可交互趋势卡）
- `article-artistic-svg.html` — 艺术 × SVG 时间线（动画里程碑）
- `article-chinese-ink-svg.html` — 国风 × SVG 时间线（印章里程碑）
- `article-minimal-svg.html` — 极简 × SVG 滚动触发（干净揭示序列）
- `article-tech-svg.html` — 科技 × SVG 进度条（数据仪表盘）

### 3. SVG 交互

嵌入公众号文章的交互式 SVG 组件。微信禁用 JavaScript，所有交互必须使用声明式 SVG 特性：SVG 内 CSS 动画、SMIL 动画、`<a>` 链接。

**SVG 流程：**
1. 阅读 `references/wechat-svg-guide.md` 了解交互模式和约束
2. 识别用户请求的交互类型
3. 从 `assets/templates/svg/` 中选择对应 SVG 模板
4. 自定义内容、颜色、时序和尺寸
5. 将 SVG 内联嵌入文章 HTML
6. 用 `scripts/validate_mp.py` 校验

**可用 SVG 模板（`assets/templates/svg/`）：**
- `svg-click-reveal.html` — 可点击跳转外链的揭示卡片
- `svg-click-hotspot.html` — 图片多点热区点击（图片地图）
- `svg-expand-cards.html` — 自动展开卡片叠（opensvg 风格，逐张展开）
- `svg-carousel.html` — SMIL 自动轮播图
- `svg-accordion.html` — 自动展开手风琴面板
- `svg-scroll-trigger.html` — 滚动触发 CSS 动画
- `svg-timeline.html` — 带动画的时间线
- `svg-comparison.html` — 前后对比滑块
- `svg-progress.html` — 动画进度条/倒计时

### 4. 校验

交付前运行校验器：

```bash
python scripts/validate_mp.py <article.html>
```

校验检查项：
- 无外部 CSS/JS 引用（`<link>`、`<script src>`）
- 无 JavaScript（`<script>`、`onclick`、`onload` 等）
- 全部内联样式（`style=""` 属性）
- 无不受支持的 HTML 标签
- 图片 `src` 为绝对 URL（非本地路径或超 32KB 的 base64）
- 字体声明为系统安全字体
- SVG 元素符合微信限制
- 文章长度在合理范围内

## 微信环境约束速查

| 约束 | 规则 |
|---|---|
| CSS | 仅内联 `style=""`。大多数情况下禁用 `<style>` 块 |
| JavaScript | 完全禁止。使用 CSS 动画或 SVG SMIL |
| 外部文件 | 不可引用外部 CSS/JS/字体 |
| 图片 | 必须托管（绝对 URL）。单张最大约 10MB |
| 字体 | 仅系统字体：PingFang SC、Microsoft YaHei、SimSun 等 |
| 表格 | 允许，常用于布局 |
| `<iframe>` | 禁止 |
| `<video>` | 支持有限，优先使用腾讯视频嵌入 |
| 文章最大体积 | ~2MB HTML。长文建议用图片长图布局 |
| 深色模式 | 微信自动反转颜色，需测试明暗双模式 |

## 交互式引导协议（强制执行）

**关键：构建任何文章之前，必须执行本协议。不得跳过问题或假设默认值。使用 `AskUserQuestion` 工具。**

### 第一阶段：基础信息（首批提问）

同时在第一条消息中提出以下 4 个问题：

1. **文章标题和作者？**
2. **文章内容准备好了吗？** 选项：
   - (A) 我已经有完整文本，直接给你
   - (B) 给我一个链接，帮我提取内容
   - (C) 先聊聊结构和风格，内容后面再补
3. **内容类型？** 选项：
   - 品牌发布 / 新品上市
   - 行业报告 / 数据分析 / 白皮书
   - 活动回顾 / 现场记录
   - 产品测评 / 使用指南
   - 深度观点 / 人物访谈
   - 生活方式 / 探店旅行
   - 艺术文化 / 摄影 / 展览
   - 其他（自由输入）
4. **期望的排版风格？** 选项：
   - 时尚杂志风（高对比，大图，crimson accent）— 适合穿搭/美妆/奢品
   - 潮流街头风（霓虹紫色，渐变色块，年轻化）— 适合潮玩/音乐/Z世代
   - 艺术画廊风（衬线字体，留白，典雅）— 适合摄影/展览/文化评论
   - 国风墨韵（宣纸色底，朱砂红章，楷体）— 适合历史/文学/传统文化
   - 极简商务风（纯黑白，零装饰）— 适合深度分析/白皮书
   - 科技数据风（统计图表，蓝色系，结构化）— 适合SaaS/数据报告
   - 温暖生活风（暖色调，柔和圆角）— 适合美食/旅行/日常
   - 奢华高端风（金色accent，暗色渐变）— 适合高端品牌/VIP内容
   - 默认编辑风（干净耐读，不挑内容）
   - 🎨 描述你想要的氛围（我会自动匹配最合适的风格）
   - 不确定，帮我推荐

**若用户选择"描述你想要的氛围"**：不要追问。解析氛围描述，匹配最接近的风格。示例：
- "深夜情感号那种感觉" → 温暖生活风 或 艺术画廊风
- "很酷的未来感" → 科技数据风
- "像Vogue杂志那样" → 时尚杂志风
- "像故宫文创那种" → 国风墨韵
- "B站科技UP主的感觉" → 潮流街头风
- "性冷淡高级感" → 极简商务风
- "江南水乡文人气息" → 国风墨韵

匹配后确认："根据你描述的___，我推荐 **某某风格**。这个风格的特点：___。合适吗？"

第一阶段完成后：匹配风格，进入第二阶段。

### 第二阶段：内容结构（第一阶段后提问）

同时提出以下 3-4 个问题：

5. **文章结构大致是什么？** 选项：
   - 标准叙事流（引言→主体→结尾）
   - 分段卡片式（多个并列板块）
   - 问答对话式
   - 时间线/里程碑式
   - 数据报告式（指标→分析→结论）
   - 纯长图（一张或多张长图滚动）
   - 其他（自由描述）

6. **有没有配图？** 选项：
   - 有很多配图，每个板块都放
   - 有几张关键图，放在特定位置
   - 只有一张封面/头图
   - 暂时没有，先用纯文字排版
   - 需要你帮我找合适的图（描述需求）

7. **配图放在哪些板块？** — 如有配图，指定位置（如"开头放封面 → 第二个板块放产品图 → 结尾放二维码"）

8. **有没有特殊板块需要突出？** 选项：
   - 有数据/指标需要突出展示
   - 有引用/金句需要视觉强化
   - 有CTA/转化按钮
   - 有品牌信息/logo需要固定位置
   - 暂时没有

### 第三阶段：交互与收尾（第二阶段后提问）

最后两个问题：

9. **需要SVG交互组件吗？** 选项：
   - 需要轮播图（适合产品展示/lookbook）
   - 需要点击揭示卡片（适合活动介绍/彩蛋，可跳转链接）
   - 需要图片热区点击（一张大图上多个可点区域，跳转不同页面）
   - 需要自动展开卡片（适合功能亮点/步骤介绍，逐张展开）
   - 需要时间线动画（适合发展历程/里程碑）
   - 需要滚动触发动画（适合列表/要点逐条揭示）
   - 需要前后对比（适合before/after效果）
   - 需要进度条/数据仪表盘（适合数据报告）
   - 不需要SVG，纯图文排版即可

10. **文章最后需要什么收尾？** 选项：
    - 引导关注/二维码
    - 相关阅读链接
    - 下期预告
    - 品牌信息/footer
    - 自然收尾，无特殊元素
    - 全部都要

## 交付与导入

生成 HTML 后，指导用户如何导入微信公众号后台。按可靠性排序展示以下方法。

### 方法一：源代码粘贴（推荐，最可靠）

用户操作步骤：
1. 用任意文本编辑器（记事本/VS Code）打开生成的 `.html` 文件
2. 全选（`Ctrl+A`），复制（`Ctrl+C`）
3. 微信公众号后台 → 新建图文 → 点击工具栏 **「源代码」** 按钮（HTML 模式，图标形如 `<>`）
4. 将全部 HTML 粘贴（`Ctrl+V`）进源代码面板
5. 再次点击源代码按钮，切回可视化模式
6. **重要：** 先将图片上传到微信素材库，再将 HTML 中图片 `src` 替换为微信托管 URL
7. 手机上预览后再发布

交付时明确告知用户：
- "打开 HTML 文件，全选复制 → 微信后台点「源代码」→ 粘贴 → 切回可视化模式"
- 提醒图片："配图的 src 目前是占位 URL，需要你先上传到微信素材库，再替换"

### 方法二：浏览器渲染后复制

用户操作步骤：
1. 在 Chrome/Edge 浏览器中打开 HTML 文件
2. 文章会以微信中的效果渲染
3. 从上到下选中全部渲染内容
4. 复制 → 直接粘贴到微信编辑器（普通模式，非源代码模式）
5. 部分 CSS 可能在传输中丢失，需检查调整

**适用场景：** 用户觉得源代码模式太复杂。复杂布局和 SVG 不如方法一可靠。

### 方法三：第三方编辑器导入（135编辑器/秀米）

用户操作步骤：
1. 复制生成的 HTML
2. 在 135编辑器 或 秀米中使用「导入文章」或「HTML导入」功能
3. 粘贴 HTML
4. 从第三方工具导出到微信

**适用场景：** 用户已在使用这些工具。多一步但提供可视化预览。

### 方法四：微信 API（进阶，面向开发者）

如果用户有公众号开发者权限：
- 使用[草稿箱 API](https://developers.weixin.qq.com/doc/offiaccount/Draft_Box/Add_draft.html) 程序化创建草稿
- 需要 `access_token` 和 API 凭证
- 作为选项提及，不主动实现

### 图片处理清单（交付前必读）

在用户复制到微信之前，提醒：

| 项目 | 操作 |
|---|---|
| 封面图 | 在微信后台单独上传，推荐 900×500px |
| 正文配图 | 先将所有图片上传到微信素材库，再替换 `src` URL |
| 图片托管 | 生成的 HTML 使用占位 URL。微信不支持外部图片热链 |
| SVG 图片 | 内联 SVG（`<svg>` 标签内）无需托管。SVG 内的 `<image href="...">` 仍需托管 URL |
| 二维码 | 如需在页脚放二维码，上传为普通图片即可 |
